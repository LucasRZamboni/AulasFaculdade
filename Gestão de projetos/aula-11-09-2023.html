Gestão de Projetos

II - Integração do Projeto

Objetivo: É uma área que serve como maestra de todas as outras áreas, orquestrando os resultados de cada uma das outras áreas em prol do projeto.

Ela coordena os objetivos de todas as outras áreas, de modo que nenhuma delas se sobressaia

O foco do PMBOK é o Projeto

Como inicia: Governança decide se o projeto vai começar ou não. A governança está avaliando, ou seja, decidindo. Depois é o gerente que toma a frente do projeto, principalmente para planejar o projeto.

Patrocinador: é quem está acima do gerente, vai dizer alguma informação estratégica para o gerente, ex. quanto dinheiro terá para o projeto.

O gerente vai ter o termo de abertura TAP/ Project Charter para poder iniciar o projeto

Termo de abertura do Projeto

É um documento estratégico elaborado pelo "Patrocinador" - podendo ser, diretor, CEO.

    Cumpre pelo menos 3 funções

  • Autorizar o início formal do projeto
  • Nomear o gerente de projeto e seus poderes e deveres. Dependendo da empresa, pode ser o Scrum Master.
  • Definir porque o projeto vai acontecer. Business Case

ERRADO: Se escrever coisas fora do nível estratégico. Pois é o nível estratégico que vai garantir o recurso para o projeto. Erra ao escrever coisas operacionais no TAP. Não pode ter qualquer coisa que seja detalhado demais.

III - Partes interessadas - Stakeholders

Qualquer pessoa, organização, departamento e etc, que possa ser afetado negativa ou positivamente pelos resultados do projeto

Não é somente uma pessoa física, pode ser uma cidade, no caso de fazer uma ponte entre uma cidade e outra. Fornecedores, empresas, terceiros, departamentos.

Pode ser positivamente ou negativamente, de qualquer parte citada a cima.

    Exemplos

  • Cliente
  • Usuário
  • Gerente
  • Patrocinador
  • Equipe
